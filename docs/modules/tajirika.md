# Tajirika — TAJIRI Partner Program

## Tanzania Context

Tanzania's service economy runs on skilled professionals — mafundi (artisans), wakili (lawyers), madaktari (doctors), wapishi (chefs), wasusi (hairstylists), wataalamu wa ngozi (skincare specialists), madalali wa nyumba (housing agents), and many more. Finding reliable professionals is word-of-mouth, quality is unpredictable, and there is no unified verification or trust system.

Tajirika ("get rich" / "prosper" in Swahili) is **TAJIRI's partner program** — the system through which skilled professionals join the platform, get verified, and become available to serve customers across TAJIRI's domain-specific modules.

**Tajirika is NOT a standalone marketplace.** It is the supply-side registration, verification, and management hub. Partners enrolled through Tajirika surface in consumer-facing modules where job matching, booking, payments, and reviews happen.

## How It Works

```
┌─────────────────────────────────────────────────────┐
│                    TAJIRIKA                          │
│           (Partner Program Hub)                      │
│                                                     │
│  Registration → Verification → Certification → Tier │
│  Portfolio    → Training      → Referrals    → Analytics │
└──────┬─────┬──────┬──────┬──────┬──────┬──────┬─────┘
       │     │      │      │      │      │      │
  ┌────▼─┐┌──▼──┐┌──▼──┐┌─▼──┐┌──▼──┐┌──▼──┐┌──▼───┐
  │Mafundi││Hair ││Skin ││Law-││Hous-││Doct-││Events│ ...
  │      ││& Na-││Care ││yer ││ ing ││ or  ││      │
  └──────┘│ils  │└─────┘└────┘└─────┘└─────┘└──────┘
          └─────┘
  (Each module owns: job matching, booking,
   payments, chat, reviews)
```

1. **Professional registers** through Tajirika — selects skill categories, submits ID, uploads portfolio
2. **Tajirika verifies** — NIDA check, TIN check, professional license/VETA certificate validation
3. **Partner is certified** — assigned tier (Mwanafunzi/Mtaalamu/Bingwa), gets verification badge
4. **Partner appears** in relevant consumer modules — a verified plumber shows up in Mafundi, a lawyer in Lawyer module
5. **Domain modules handle** job matching, booking, service delivery, payments, and reviews
6. **Tajirika aggregates** — partner sees all earnings, ratings, and performance across modules in one dashboard

## Feature List

1. Multi-step registration flow with personal info, skill selection, ID submission, portfolio upload
2. Support for multiple skill categories per partner (e.g., a fundi who does plumbing AND electrical)
3. National ID (NIDA) number submission for identity verification
4. TIN number submission for tax compliance
5. Professional license/certificate upload (VETA, TLS, Medical Council, etc.)
6. Portfolio upload — photos/videos of past work with descriptions
7. Service area definition (region/district/ward level using Tanzania administrative hierarchy)

9. Background check consent and processing

### Identity & Credential Verification
11. NIDA number verification via government API
12. TIN verification via TRA integration
13. VETA certificate verification for trade professionals

## What Domain Modules Handle (NOT Tajirika)

These are explicitly **out of scope** for Tajirika — each consumer module owns:

- **Job/service posting** — customers create requests within the domain module
- **Matching algorithm** — domain-specific matching (proximity for Mafundi, specialization for Lawyer, etc.)
- **Booking & scheduling** — appointment/job booking flows
- **In-service communication** — job-specific chat between customer and partner
- **Payments & escrow** — service-specific payment flows through WalletService
- **Ratings & reviews** — domain-specific review criteria and display
- **Service-specific workflows** — consultation flow for Doctor, case management for Lawyer, project milestones for Mafundi

Domain modules **query Tajirika** to:
- Get verified partners by skill category and service area
- Check partner verification status and tier
- Display partner credentials and portfolio
- Report job completion (feeds back into Tajirika analytics and tier progression)

## TAJIRI Integration Points

### ProfileService (`lib/services/profile_service.dart`)

Tajirika extends the user's TAJIRI profile with partner-specific data. The existing `FullProfile` model (`lib/models/profile_models.dart`) includes `stats` (ProfileStats with postsCount, friendsCount, followersCount, subscribersCount), `location` (ProfileLocation with region/district/ward/street), and `friendshipStatus`. Tajirika adds partner verification badge, tier level, skill categories, and aggregate rating to this profile.

- `ProfileService.getProfile(userId)` — Domain modules call this to check if a user is a verified Tajirika partner and display their tier badge
- `ProfileService.updateProfile(data)` — Used during registration to update the user's profile with partner information
- The `ProfileScreen` (`lib/screens/profile/profile_screen.dart`) already supports configurable tabs via `ProfileTabConfig` — Tajirika adds a "Partner" tab showing skills, portfolio, and credentials

### WalletService (`lib/services/wallet_service.dart`)

All partner earnings flow through the existing TAJIRI wallet. The `Wallet` model (`lib/models/wallet_models.dart`) tracks `balance`, `pendingBalance`, `currency` (TZS), `isActive`, `hasPin`, and `adBalance`. `WalletTransaction` tracks individual transactions with `type` (deposit/withdrawal/transfer_in/transfer_out/payment/refund), `amount`, `fee`, `balanceBefore`, `balanceAfter`, `status`, `paymentMethod`, `provider`.

- Payout withdrawals to M-Pesa/Tigo Pesa/Airtel Money via wallet withdrawal flow
- Referral bonuses credited as wallet deposits
- Aggregated earnings overview pulls from `WalletTransaction` history filtered by partner-related transaction types
- The existing `WalletScreen` and `EarningsDashboardScreen` (`lib/screens/wallet/`) provide the earnings UI foundation

### LocationService (`lib/services/location_service.dart`)

Service area definition uses Tanzania's administrative hierarchy. The existing `location_models.dart` defines `Region` (id, name, postCode), `District` (id, regionId, name, postCode), `Ward` (id, districtId, name, postCode), `Street` (id, wardId, name), and `UserLocation` (region, district, ward, street).

- `LocationService.searchLocations()` — Partners define service areas at region/district/ward level during registration
- Domain modules use these locations to match customers with nearby partners
- The existing `LocationPicker` widget (`lib/widgets/location_picker.dart`) provides the selection UI for region → district → ward → street

### FCMService + NotificationService (`lib/services/fcm_service.dart`, `lib/services/notification_service.dart`)

Push notifications for partner events. The existing `AppNotification` model (`lib/models/notification_models.dart`) supports types including `payment`, `wallet`, `system`, and others. FCMService handles Firebase Cloud Messaging with payload-based routing.

- Verification status changes (NIDA verified, TIN verified, license approved)
- Tier upgrades (promoted to Mtaalamu, promoted to Bingwa)
- New referral signups and bonus credits
- Training reminders and course completions
- Re-verification expiry alerts
- Account health warnings (rating drop, inactive status)

### PhotoService + VideoUploadService (`lib/services/photo_service.dart`, `lib/services/video_upload_service.dart`)

Portfolio media management. The existing `Photo` model (`lib/models/photo_models.dart`) tracks photos with `filePath`, `thumbnailPath`, `width`, `height`, `caption`, `locationName`, organized in `PhotoAlbum` collections with `privacy` settings.

- `PhotoService.uploadPhoto()` — Portfolio photo uploads (before/after work photos, credentials)
- `VideoUploadService` — Video showcase uploads (work demonstrations, introductions), uses Dio for chunked upload with 10-min timeout
- Portfolio photos organized in albums per skill category
- The existing `PhotosScreen` and `AlbumDetailScreen` (`lib/screens/photos/`) provide album management UI

### PostService + StoryService (`lib/services/post_service.dart`, `lib/services/story_service.dart`)

Partners share work publicly for visibility. The existing `Post` model supports types (text, photo, video, audio, shortVideo) with `media[]`, `hashtags[]`, `locationName`, `taggedUsers[]`, engagement metrics (likesCount, commentsCount, sharesCount, viewsCount).

- Partners share completed work as portfolio posts with before/after photos
- Daily work stories via StoryService for visibility and marketing
- Trade-specific hashtags (#FundiUmeme, #Useremala, #WakiliDar) via `HashtagService` for discovery
- The existing `CreatePostScreen` handles media-rich post creation

### GroupService (`lib/services/group_service.dart`)

Trade-specific partner communities. The existing `Group` model (`lib/models/group_models.dart`) includes `name`, `slug`, `description`, `privacy` (public/private/secret), `membersCount`, `postsCount`, `rules[]`, `conversationId` for group chat.

- Trade-specific groups (e.g., "Mafundi wa Umeme Dar es Salaam", "Wakili wa Dar es Salaam", "Wasusi wa Dar")
- Knowledge sharing, peer support, job referrals within trade groups
- Mentorship connections within groups — Bingwa partners mentor Mwanafunzi
- Group events for training workshops and trade meetups
- The existing `GroupsScreen` and `GroupDetailScreen` (`lib/screens/groups/`) handle group management

### EventService (`lib/services/event_service.dart`)

Training workshops and trade events. The existing `EventModel` (`lib/models/event_models.dart`) supports `name`, `description`, `startDate`, `endDate`, `locationName`, `isOnline`, `onlineLink`, `privacy`, `category`, `ticketPrice`, associated `group`.

- `EventService.createEvent()` — Create training workshops, VETA partnership courses, trade meetups
- `EventService.getUserEvents()` — Track enrolled training sessions
- `EventService.respondToEvent()` — RSVP to training events
- Partner training courses created as events within trade groups

### EventTrackingService (`lib/services/event_tracking_service.dart`)

Partner journey analytics. The existing `UserEvent` model (`lib/models/flywheel_models.dart`) tracks `eventType`, `postId`, `creatorId`, `timestamp`, `durationMs`, `sessionId`, `metadata{}`.

- `EventTrackingService.trackEvent()` — Track: registration funnel steps, verification submissions, training engagement, portfolio uploads, referral link shares
- Partner performance metrics: jobs completed across modules, response time, tier progression
- Aggregated analytics for the Partner Dashboard

### PeopleSearchService (`lib/services/people_search_service.dart`)

Partner discovery by domain modules. The existing `PersonSearchResult` model (`lib/models/people_search_models.dart`) includes `regionName`, `districtName`, `employer`, `mutualFriendsCount`, `friendshipStatus`, `isOnline`.

- Domain modules search for partners by skill category, service area, tier, and rating
- Tajirika registers partner skill metadata in the search index
- The existing `UniversalSearchScreen` (`lib/screens/search/`) supports user search — extended for partner-specific filters

### PresenceService (`lib/services/presence_service.dart`)

Partner online/available status. PresenceService provides batch online status checking.

- Show partner availability status (available/busy/offline) in domain module listings
- Customers see which partners are currently active for urgent service requests
- Status synced across all domain modules where the partner appears

### CalendarService (`lib/services/calendar_service.dart`)

Booking aggregation. `CalendarService.createEvent()` creates calendar entries.

- Aggregated booking calendar from all domain modules where partner has active bookings
- Availability schedule management — partners set available hours
- Training session reminders synced to personal calendar
- The existing `EventsScreen` (`lib/screens/events/`) provides calendar UI foundation

### LiveUpdateService (`lib/services/live_update_service.dart`)

Real-time updates via Firebase Firestore listeners. Uses sealed classes: `FeedUpdateEvent`, `PostUpdateEvent`, `ProfileUpdateEvent`, `MessagesUpdateEvent`, etc.

- `ProfileUpdateEvent` triggers when partner tier or rating changes across any module
- Real-time verification status updates (NIDA check completed, license approved)
- New referral notifications
- Domain modules fire live updates when jobs are completed, triggering Tajirika analytics refresh

### MessageService (`lib/services/message_service.dart`)

Partner support channel. The existing `Conversation` model supports types (private, group, broadcast) with `participants[]`, `lastMessage`, `unreadCount`.

- `MessageService.sendMessage()` — Dedicated partner support conversation
- Dispute escalation messages from domain modules routed to partner support
- The existing `ChatScreen` (`lib/screens/messages/chat_screen.dart`) handles messaging UI

### ContentEngineService (`lib/services/content_engine_service.dart`)

Partner profile discovery. The existing content engine supports `search()` with `types[]`, `category`, `region`, `sort` parameters.

- Partner profiles discoverable through content recommendations
- Trade-specific content surfaced in feeds (e.g., portfolio posts from local fundis)
- `ContentEngineService.search()` with partner-specific type filters

### SubscriptionService (`lib/services/subscription_service.dart`)

Partner premium features. The existing `SubscriptionTier` model includes `name`, `price`, `billingPeriod`, `benefits[]`.

- Premium partner tiers with enhanced visibility, reduced commission, priority matching
- Subscription benefits displayed on partner profile

### Shared Data Model (Tajirika as source of truth)

```
TajirikaPartner {
  userId          → links to TAJIRI FullProfile (lib/models/profile_models.dart)
  skills[]        → list of SkillCategory with specializations
  tier            → Mwanafunzi | Mtaalamu | Bingwa (PartnerTier enum)
  verifications {
    nida          → pending | verified | failed (NIDA number)
    tin           → pending | verified | failed (TRA TIN number)
    professional  → per-domain license status (VETA, TLS, Medical Council, etc.)
    background    → pending | cleared | flagged
  }
  serviceArea {
    regions[]     → Region ids (lib/models/location_models.dart)
    districts[]   → District ids
    wards[]       → Ward ids
  }
  portfolio[]     → Photo/Video references (PhotoAlbum via lib/models/photo_models.dart)
  training {
    completed[]   → TrainingCourse ids with certificates
    enrolled[]    → in-progress courses
  }
  aggregateRating → weighted average from all domain modules
  jobsCompleted   → total across all domain modules
  responseTime    → average response time to job requests
  referralCode    → unique referral identifier
  referrals[]     → people referred and their status
  payoutAccount   → M-Pesa/Tigo Pesa/Airtel Money/bank account
  isActive        → partner status on platform
  createdAt       → registration date
}
```

### API Contract

Domain modules query Tajirika for verified partners:
```
GET /tajirika/partners?skills=plumbing,electrical&region=dar_es_salaam&tier=mtaalamu&min_rating=4.0&available=true
```

Domain modules report job completions back to Tajirika:
```
POST /tajirika/partners/{id}/job-completed
{
  "module": "mafundi",
  "jobId": "...",
  "rating": 4.8,
  "earnings": 50000,
  "completedAt": "2026-04-08T14:30:00Z"
}
```

Tajirika checks tier eligibility:
```
GET /tajirika/partners/{id}/tier-progress
→ { currentTier: "mwanafunzi", nextTier: "mtaalamu", requirements: { jobsNeeded: 4, ratingNeeded: 0, verificationsPending: ["veta"] } }
```
