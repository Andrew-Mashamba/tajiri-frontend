# My Baby — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Complete the My Baby postnatal tracker with all 57 features from the design spec.

**Architecture:** Flutter module with backend API, SQLite local cache for offline-first. Key design principle from Huckleberry: ONE-SECOND access to "add feeding" button — sleep-deprived parents need zero friction.

**Tech Stack:** Flutter/Dart, Laravel backend, PostgreSQL, SQLite (sqflite), FCM

**Spec:** `docs/modules/my_baby.md`

## Task 2: Fix Existing Pages — Auth, Bilingual, Bugs

**What:** Fix the 4 critical bugs in existing code before building new features.

### 2a. Add auth token to ALL service calls

**File:** `lib/my_baby/services/my_baby_service.dart`

**Problem:** Every HTTP call uses bare headers `{'Content-Type': 'application/json'}` — no Bearer token. Authenticated endpoints will 401.

**Fix:** Add `required String token` parameter to every method. Use `ApiConfig.authHeaders(token)` for all requests. Follow the pattern used by every other service in the codebase.

```dart
// BEFORE (broken):
final response = await http.post(
  Uri.parse('$_baseUrl/my-baby/babies'),
  headers: {'Content-Type': 'application/json'},
  body: jsonEncode({...}),
);

// AFTER (fixed):
final response = await http.post(
  Uri.parse('$_baseUrl/my-baby/babies'),
  headers: ApiConfig.authHeaders(token),
  body: jsonEncode({...}),
);
```

**Then update all callers** (my_baby_home_page.dart, baby_dashboard_page.dart, feeding_tracker_page.dart, vaccination_page.dart) to read token from `LocalStorageService` and pass it through.

### 2b. Add bilingual text to all pages

### 2c. Fix vaccination persistence bug

**File:** `lib/my_baby/pages/vaccination_page.dart`

**Problem:** Lines 96-117 — when using the default schedule (locally generated, id < 100), marking a vaccination as done only updates local state via `setState`. On refresh or page re-entry, all marks are lost.

**Fix:** Two-part fix:
1. **Immediate:** When baby is registered, backend should auto-create the Tanzania EPI schedule and return real vaccination records with real IDs. Remove the `_buildDefaultSchedule()` fallback entirely.
2. **Fallback:** If backend hasn't seeded yet, persist local marks to SQLite (Task 13) so they survive refresh.

### 2d. Fix milestone toggle can't undo

**File:** `lib/my_baby/pages/baby_dashboard_page.dart`, line 493

**Problem:** `if (milestone.isDone) return;` — once marked done, tapping does nothing. No way to undo.

**Fix:** Add `unmarkMilestoneDone` to service. On tap of completed milestone, show confirmation dialog ("Ondoa alama?" / "Remove mark?"), then call uncomplete endpoint.

### 2e. Fix vaccination schedule completeness

**File:** `lib/my_baby/pages/vaccination_page.dart`, `_buildDefaultSchedule()`

**Problem:** Missing vaccines from Tanzania EPI: IPV at 14 weeks, Vitamin A at 6/9/12/18 months, DPT-HepB-Hib booster at 18 months.

**Fix:** Add the missing entries to `_buildDefaultSchedule()`. This is a stopgap until backend seeds them (Task 1).

Duration _wakeWindow(int ageMonths) {

## Task 15: SQLite Offline Cache

**What:** New file `lib/my_baby/services/my_baby_database.dart` — SQLite local-first cache.

**Following the pattern from `SQLITE_ADOPTION_ROADMAP.md`:**

**Tables:**
```sql
CREATE TABLE babies (id INTEGER PRIMARY KEY, user_id INTEGER, json_data TEXT, updated_at TEXT);
CREATE TABLE feeding_logs (id INTEGER PRIMARY KEY, baby_id INTEGER, json_data TEXT, logged_at TEXT);
CREATE TABLE sleep_sessions (id INTEGER PRIMARY KEY, baby_id INTEGER, json_data TEXT, start_time TEXT);
CREATE TABLE diaper_logs (id INTEGER PRIMARY KEY, baby_id INTEGER, json_data TEXT, logged_at TEXT);
CREATE TABLE growth_measurements (id INTEGER PRIMARY KEY, baby_id INTEGER, json_data TEXT, measured_at TEXT);
CREATE TABLE health_logs (id INTEGER PRIMARY KEY, baby_id INTEGER, json_data TEXT, start_date TEXT);
CREATE TABLE vaccinations (id INTEGER PRIMARY KEY, baby_id INTEGER, json_data TEXT);

-- Pending mutations queue
CREATE TABLE pending_mutations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  entity_type TEXT,      -- 'feeding', 'sleep', 'diaper', etc.
  mutation_type TEXT,     -- 'create', 'update', 'delete'
  json_data TEXT,         -- full request body
  retry_count INTEGER DEFAULT 0,
  created_at TEXT
);

-- Sync state
CREATE TABLE sync_state (
  entity_type TEXT PRIMARY KEY,
  last_synced_at TEXT,
  last_synced_id INTEGER
);
```

**Offline-first flow:**
1. User taps "Log feeding" → write to SQLite immediately → UI updates
2. If online → send to backend → on success, update SQLite record with server ID
3. If offline → insert into `pending_mutations` → show sync icon in UI
4. On connectivity restored → process pending_mutations queue in order → retry up to 3 times

**Sync service** (`lib/my_baby/services/my_baby_sync_service.dart`):
```dart
class MyBabySyncService {
  // Called on app launch + connectivity change
  static Future<void> syncAll(String token) async {
    await _processPendingMutations(token);
    await _pullRemoteChanges(token);
  }
  
  static Future<void> _processPendingMutations(String token) async {
    final pending = await MyBabyDatabase.getPendingMutations();
    for (final mutation in pending) {
      try {
        await _sendMutation(mutation, token);
        await MyBabyDatabase.removePendingMutation(mutation.id);
      } catch (e) {
        await MyBabyDatabase.incrementRetryCount(mutation.id);
      }
    }
  }
}
```

**This is CRITICAL** — parents log at 3 AM with spotty WiFi. Every write must succeed locally.

**Files to create:**
- `lib/my_baby/services/my_baby_database.dart` — SQLite schema + CRUD
- `lib/my_baby/services/my_baby_sync_service.dart` — sync logic

**Verify:** Put device in airplane mode. Log a feeding, a sleep session, and a diaper change. All appear in UI. Turn airplane mode off. Data syncs to backend. Confirm with direct API call.
