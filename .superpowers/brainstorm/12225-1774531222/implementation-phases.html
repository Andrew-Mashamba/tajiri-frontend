Section 6: Implementation Phases

Ship in 4 phases — each phase delivers standalone value

Phase 1: Foundation & Tracking

BUILD FIRST

Invisible to users. Lays the data pipeline everything else depends on.

BE
user_events table + POST /api/events endpoint

Migration, model, controller. Accepts batched events. Partition strategy.

FE
EventTrackingService

Dart service: captures view, dwell, scroll_past, like, share, save events. Batches locally, flushes every 30s or on app background. Integrates into PostCard, FullScreenPostViewer, FeedScreen.

BE
user_interest_profiles table + Claude CLI profile builder

Scheduled job every 15 min. Sends user event batch to Claude CLI. Stores computed interest profile. Redis cache layer.

BE
creator_streaks + viewer_streaks tables

Migrations, models. Streak calculation logic. Updated on post creation (creator) and app open (viewer). Bank/freeze logic.

BE
creator_scores table + weekly calculation job

Score formula: community (reply rate) + quality (engagement rate) + consistency (posting frequency). Tier assignment. GET /api/creators/{id}/score endpoint.

Ship gate: Events flowing, profiles computing, streaks tracking. No user-visible changes yet.

Phase 2: Gossip Engine + Personalized Feed

CORE VALUE

First user-visible changes. Feed gets smarter, gossip threads appear.

BE
gossip_threads + thread_title_templates tables

Migrations, models. Seed ~50 bilingual title templates. Thread CRUD. GET /api/gossip/threads and /threads/{id} endpoints.

BE
Thread detection job (Claude CLI, every 5 min)

Velocity spike detection. Claude clusters related posts. Template + slot assignment. Thread lifecycle (active → cooling → archived).

BE
Personalized feed ranking (Claude CLI)

GET /api/feed/personalized. Sends candidate posts + user interest profile to Claude. Returns ranked post_ids. 5-min cache per user. Fallback to trending_score sort.

FE
GossipThreadCard widget + ThreadViewerScreen

Thread card injected into feed every 8-12 posts. Thread viewer: title, seed post, related posts, live counter, swipe between threads. Post model gains thread_id/thread_title fields.

FE
Feed switches to personalized endpoint

FeedScreen and DiscoverFeedContent use /api/feed/personalized. Trending tab shows threads first. Existing discover/nearby unchanged as fallback.

FE
Thread indicator on PostCard

Small badge on posts that are part of a gossip thread. Tap opens thread viewer.

Ship gate: Gossip threads appearing, feed personalized, users can browse and enter threads.

Phase 3: Addiction Loops + Creator Payments

RETENTION

Hooks that keep users coming back and creators posting. The Flywheel starts spinning.

BE
creator_fund_pools + creator_fund_payouts tables

Monthly pool config (thresholds at 0 for testing). Distribution job on 1st of month. Multiplier calculation: tier × streak × community × virality. Full transparency in payout records.

BE
notification_templates table + digest/FOMO push notifications

Bilingual templates seeded. Morning/evening digest job (Claude CLI selects threads per user). FOMO trigger job (thread velocity spike → push to relevant users, max 3/day). Streak warning notifications.

BE
GET /api/gossip/digest + weekly report endpoints

Digest: personalized thread selection + proverb of the day. Weekly report: earnings summary, best post, trend arrows, engagement comparison.

FE
DigestScreen + WeeklyReportScreen

Digest: proverb, thread summaries, unfinished threads, one-tap navigation. Weekly report: earnings card, best post, trend arrows, tip of the week.

FE
StreakIndicatorWidget + CreatorTierBadge + MilestoneOverlay

Flame streak on profile. Tier badge on profile + post cards. Full-screen milestone celebration. Integrated into ProfileScreen and CreatorStatsPanel.

FE
Enhanced Creator Stats Panel + Earnings Dashboard

Existing panels gain: tier badge, streak multiplier display, community bonus indicator, virality bonus, fund pool share projection, each multiplier shown transparently.

FE
FCM notification routing for new types

digest → DigestScreen, thread_trending → ThreadViewerScreen, streak_warning → CreatePostScreen, weekly_report → WeeklyReportScreen, milestone → MilestoneOverlay.

FE
Rabbit hole mechanics in feed

Teaser cards between posts ("A creator you follow just went viral"). Autoplay chain enhancements (Up Next preview, reduced swipe friction after 3+ posts). Invisible depth milestones (better content at 10+, thread recommendation at 25+).

Ship gate: Flywheel is spinning. Creators earning, viewers hooked, gossip driving engagement.

Phase 4: Advanced + Optimization

SCALE

Polish, advanced features, and monetization expansion. Flywheel accelerates.

Sponsored posts marketplace

Business-to-creator matching. Sponsored post creation flow. "Sponsored" tag on posts. Commission handling. Star/Legend tier only.

Collaboration radar

Claude CLI detects creator pairs in similar niches with growing audiences. Suggests collaborations. 2x algorithm boost for collab posts.

Intensity graduation system

Per-user addiction intensity level (gentle → medium → full) based on engagement response. Users who respond to FOMO get more FOMO. Users who don't, stay at gentle.

Creator battles / split threads

When two creators post opposing takes, Claude creates "Side A vs Side B" thread. Users pick sides. Engagement amplifier.

Ad revenue share (when ads launch)

55/45 creator/platform split. In-feed ads, pre-roll on videos. Only on posts with 1K+ views. Creator opt-out available.

Analytics & A/B testing framework

Measure: DAU/MAU, session depth, streak retention, feed CTR, thread entry rate, creator earnings satisfaction. A/B test multiplier values, notification frequency, thread card placement.

Ship gate: Full Flywheel with all 3 revenue tiers, advanced gossip, and data-driven optimization.
