Section 5: Data Model & API Design

New tables, endpoints, and how they connect to existing infrastructure

New Database Tables

Additions to the existing Laravel backend — no changes to existing tables

user_events AI Personalization — behavioral tracking
id, user_id, event_type (view|like|share|save|scroll_past|dwell|comment|follow|unfollow), post_id?, creator_id?, duration_ms?, metadata (JSON), created_at

High-write table. Partitioned by month. Pruned after 90 days (profiles are pre-computed).

user_interest_profiles AI Personalization — computed by Claude CLI
id, user_id (unique), topic_weights (JSON), creator_affinities (JSON), format_preferences (JSON), activity_patterns (JSON), gossip_affinity (JSON), commerce_signals (JSON), updated_at

One row per user. Refreshed every 15 min by scheduled job. Cached in Redis.

gossip_threads Gossip Engine — thread metadata
id, seed_post_id, title_key, title_slots (JSON), category (entertainment|business|music|sports|politics|relationships|local), velocity_score, post_count, participant_count, status (active|cooling|archived), geographic_scope (global|national|local), location_lat?, location_lng?, location_radius_km?, created_at, updated_at, archived_at
gossip_thread_posts Gossip Engine — post-to-thread mapping
id, thread_id, post_id, relevance_score, added_at

A post can belong to multiple threads. Relevance score determines display order within thread.

thread_title_templates Gossip Engine — bilingual templates
id, key (unique), template_en, template_sw, slots (JSON array of slot names), category, tone (hot|breaking|milestone|battle|local)

Pre-written by humans. ~50-100 templates covering all categories and tones. Claude picks template + fills slots.

creator_scores Creator Payments — rolling 30-day score
id, user_id (unique), score (0-100), tier (rising|established|star|legend), community_score, quality_score, consistency_score, follower_growth_velocity, threads_triggered_30d, avg_engagement_rate_30d, calculated_at

Recalculated weekly by scheduled job. Historical snapshots in creator_score_history.

creator_streaks Addiction Engine — posting streak tracker
id, user_id (unique), current_streak_days, longest_streak_days, last_post_at, banked_skip_days, streak_frozen (bool), frozen_at?, multiplier (computed)
viewer_streaks Addiction Engine — daily open tracker
id, user_id (unique), current_streak_days, longest_streak_days, last_active_date, streak_frozen (bool)
creator_fund_pools Creator Payments — monthly pool config
id, month (YYYY-MM), total_amount, currency, distributed (bool), distributed_at?, qualification_min_followers, qualification_min_posts, qualification_min_views

Testing: min_followers=0, min_posts=1, min_views=0. Adjustable per month.

creator_fund_payouts Creator Payments — individual payout records
id, pool_id, user_id, base_score, tier_multiplier, streak_multiplier, community_multiplier, virality_multiplier, final_score, pool_share_pct, payout_amount, status (pending|paid|failed), paid_at?

Full transparency — every multiplier recorded. Creator can see exactly why they earned what they earned.

notification_templates Addiction Engine — push notification templates
id, key (unique), title_en, title_sw, body_en, body_sw, slots (JSON), category (digest|fomo|milestone|streak|earnings), max_per_day, priority

New API Endpoints

POST /api/events Log user behavior event (batch)
GET /api/feed/personalized AI-ranked feed for current user
GET /api/gossip/threads List active gossip threads (personalized, paginated)
GET /api/gossip/threads/{id} Thread detail with posts
GET /api/gossip/digest Personalized morning/evening digest
GET /api/creators/{id}/score Creator score, tier, multipliers
GET /api/creators/{id}/streak Posting streak status and multiplier
GET /api/creators/{id}/fund-payout Current month fund pool projection
GET /api/creators/{id}/weekly-report Weekly performance summary
GET /api/users/{id}/streak Viewer daily open streak
POST /api/creators/{id}/payout/request Request mobile money payout

Changes to Existing Infrastructure

Minimal — mostly additive

Feed endpoints enhanced

Existing /api/feed/discover and /api/feed/trending gain optional ?personalized=true param. When set, backend uses AI-ranked order. Fallback to current behavior when omitted. Non-breaking change.

PostCard gains thread indicator

Post model gets optional thread_id and thread_title fields. PostCard shows small "Part of trending thread" badge when present. Tap opens thread viewer.

Creator Stats Panel enhanced

Existing _CreatorStatsPanel in post_detail_screen.dart gains: tier badge, streak indicator, multiplier breakdown, gossip thread count. All data from new endpoints — no changes to existing earnings logic.

FCM notification routing extended

New notification types added to FCM payload routing: digest → DigestScreen, thread_trending → ThreadViewerScreen, streak_warning → CreatePostScreen, weekly_report → WeeklyReportScreen, milestone → celebration overlay.
